# Migration System Patches Applied

## Summary of Changes

Three critical patches have been applied to `scripts/cli/migrate.ts` to fix the migration system:

### 1. Fixed NOT NULL rewrite regex (Line 336)
**Problem**: The regex was incorrectly matching ADD COLUMN statements that included DEFAULT clauses, causing safe PostgreSQL ≥11 operations to be unnecessarily rewritten.

**Fix**: Updated the regex to exclude statements with DEFAULT clauses:
```typescript
// OLD: /ADD\s+"(\w+)"\s+((?:[^N]|N(?!OT))*?)NOT\s+NULL/gi
// NEW: /ADD\s+"(\w+)"\s+((?:[^N]|N(?!OT))*?)NOT\s+NULL(?!\s*DEFAULT)/gi
```

### 2. Added `isAutoGenerated` flag to HeaderOptions (Line 280)
**Problem**: No way to distinguish machine-generated files from manual ones for governance bypass.

**Fix**: Added `isAutoGenerated?: boolean` to the `HeaderOptions` interface.

### 3. Auto-inject governance bypass comments (Lines 345-351)
**Problem**: Machine-generated companion files required manual governance annotations.

**Fix**: Added logic to automatically inject bypass comments for:
- BASELINE_ migrations (always mixed ops by design)
- BREAKING_ migrations when `isAutoGenerated: true`

### 4. Updated call sites to use `isAutoGenerated: true` (Lines 445, 535)
**Problem**: Companion files and primary files weren't marked as auto-generated.

**Fix**: Added `isAutoGenerated: true` to both companion file headers and primary file headers.

## Result

The migration system now:
- ✅ Correctly handles ADD COLUMN ... NOT NULL DEFAULT statements without unnecessary rewrites
- ✅ Automatically bypasses governance checks for machine-generated files
- ✅ Generates clean, single-file migrations for schema alignment
- ✅ Maintains all safety checks for manual migrations
